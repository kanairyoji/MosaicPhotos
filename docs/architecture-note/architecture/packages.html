パッケージと依存関係

機能はローカル SPM パッケージ 11 個に分かれる。基盤 → ロジック → UI → 統合 → アプリ、と一方向に依存する。

各パッケージの責務

パッケージ層責務
MosaicSupport基盤横断ユーティリティ。共通ロガー LogChannel、端末診断 Diagnostics、メモリ圧迫監視、自己修復 makeResilientModelContainer、電源/回線/重い処理ゲート。依存なし。
PhotoSourceKit基盤写真ソース共通基盤。PhotoStore/PhotoItem/PhotoLoadState と共通ビュー（グリッド・全画面）。逆ジオコーディング・EXIF などの純ロジック。
ImageCacheKit基盤メモリ（MemoryImageCache）+ ディスク I/O（DiskImageStore）の画像キャッシュプリミティブ。破棄ポリシーは利用側が持つ。
LocalPhotoCoreロジック端末写真ロジック。LocalPhotoStore、アルバム走査、サムネキャッシュ。SwiftUI 非依存。
DropboxCoreロジックDropbox の認証（OAuth PKCE）・API・差分同期・キャッシュ（SwiftData + ImageCacheKit）。SwiftUI 非依存。
BackupKitロジック端末写真 → Dropbox バックアップのオーケストレーション、アップロード、People/Album インデックス。
AutoAlbumCoreロジック自動アルバム生成・知覚・意味検索（タグ台帳＋LLM 審査）・シーンタグ台帳（TagStore）・ピープル顔クラスタ（Faces）のロジック。SwiftUI 非依存・純ロジック中心でテストしやすい。共通の背景トリクル骨格 BackgroundTrickle を持つ。
LocalPhotoKitUI端末写真の UI。LocalPhotoCore を @_exported import。
DropboxKitUIDropbox の UI。DropboxCore を @_exported import。
MobileCLIPKitUI/実装CLIP/翻訳/VLM キャプション/顔モデルのランタイムと AutoAlbumCore の seam 実装（知覚・タグ・顔アダプタ・表示ラベラ）。Core ML / Vision / Foundation Models を扱う。Core ML ロードと BPE の共通足場（CoreMLModelSupport・BPESupport）を持つ。
PhotosFeatureKit統合ローカル + Dropbox を統合する MergedPhotoStore / MergedPhotoItem、場所グルーピングの PlaceScanner。

依存の向きに関する約束

  • ロジック層はアプリを import しない。DropboxCore の依存は Foundation / AuthenticationServices / CryptoKit / Security / SwiftData / UIKit / ImageCacheKit / MosaicSupport のみ。
  • UI は Core を再エクスポート（@_exported import）。アプリは import DropboxKit だけで両層の公開型を参照できる。
  • 知覚 seam はプロトコル + MobileCLIPKit 実装。AutoAlbumCore はプロトコルだけを定義し、Core ML 依存の実体は MobileCLIPKit が持つ。アプリが結線する。
設計判断 Core/UI を分けると、SwiftUI を持たない純ロジックを macOS の swift test で高速に検証できる。AI の重い依存（Core ML）を MobileCLIPKit に隔離することで、AutoAlbumCore 本体はモデルなしでもテスト可能になる。詳細は SPM モジュール分割。
共通足場の抽出と責務分割（ADR-26） 機能追加でコピーが蓄積した骨格を挙動不変で共通化した：背景トリクル（BackgroundTrickle）・自己修復コンテナ（makeResilientModelContainer）・Core ML ロード/BPE（CoreMLModelSupport/BPESupport）・UI 小物（BusyLabel/LoadingRow/sectionHeader/loadCover）。あわせて肥大した型を分割した（AIAlbumService→AIAlbumInterpreter/AIAlbumVerificationCoordinator/QueryEmbedder、BackupEngine→BackupRunner/BackupProgressStore、DropboxThumbnailBatcher→DropboxThumbnailBatchRequest/DropboxThumbnailChunkFetcher）。
