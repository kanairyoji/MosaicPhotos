データ構造詳細

アルバム・付加情報・埋め込み・キャッシュの永続モデルと、それを安全に運ぶ Sendable 値型。両者の対応関係が設計の肝。

統一キー: PhotoRef / refKey

ローカル（PHAsset.localIdentifier）とクラウド（Dropbox path）を 1 つの文字列キーに統一する。

種別encoded（refKey）
.local(id)"L-" + id
.cloud(path)"C-" + path

PhotoRef.decode で逆変換。これにより付加情報・アルバムのメンバー・埋め込みをソース横断で同一キー管理できる。

@Model と Sendable 値の対応

鉄則 @Model は @ModelActor の外へ出さない。境界をまたぐときは必ず Sendable 値型へ変換する（GeneratedAlbum.asInfo / PhotoEnrichment.asEnrichedPhoto）。UI と actor の間で参照型の競合を起こさないため。
@Model（永続・actor 内）Sendable 値（境界外）
GeneratedAlbumAutoAlbumInfo
PhotoEnrichmentEnrichedPhoto
PhotoEmbedding（ページで (refKey, Data) として取り出す）

付加情報: PhotoEnrichment

写真 1 枚のメタデータ。埋め込みは含まない（別テーブル）。

フィールド意味
refKey (unique)統一キー
kind / localIdentifier / cloudPath出所
captureDate / latitude / longitude / placeName / country時間・場所
linkKeyローカル↔クラウドの同一写真を束ねる鍵（重複排除）
people人物名（顔クラスタ由来。旧 PhotoKit People は撤去）
isScreenshot / isFavorite / aspectカバー選定・除外の判断材料
sceneTaggedCLIP 埋め込みを試行済みか（未処理抽出・無限ループ防止）
別コンテナに分けるもの シーンタグ／VLM キャプションは TagStore（TagsV1）、顔クラスタは FaceStore（FacesV1）と、AI アルバム本体（AutoAlbumV10）とは別の ModelConfiguration に分離する。スキーマ変更や破棄採番が互いのデータを壊さないため。

埋め込み: PhotoEmbedding（別テーブル・fp16）

メタデータ取得が巨大 blob に触れないよう分離。refKey(unique) + vector: Data（Float16 パック・約1KB）。読み出し時に fp32 へ復元する。理由は 事例: メモリ枯渇。

生成アルバム: GeneratedAlbum / AutoAlbumInfo

旅行・フォルダ・AI アルバムの永続レコード。Photos.app は変更しない仮想アルバム。

フィールド意味
id (unique)安定キー。strategyID:開始epoch-終了epoch（再生成時の upsert キー）
strategyID生成戦略（TimePlace / PathAlbum / AIAlbum）
memberRefs / coverRefメンバー・カバーの refKey（ローカル/クラウド混在）
places / country / startDate / endDate / latitude,longitude表示・タイトル生成用
criteriaAI アルバムの元クエリ（編集で呼び戻す）。他種別は nil

AutoAlbumInfo（Sendable・Codable）は表示用の派生プロパティを持つ: localIdentifiers/cloudPaths（メンバー分解）、placesLabel（"A & B +N"）、nights/durationLabel。Codable なので将来 Dropbox への JSON コピーにも使える。

スキーマのバージョニング

スキーマを変えたら ModelConfiguration 名を採番して旧ストアを破棄→再構築する（現行 "AutoAlbumV10"）。マイグレーションを書かず破棄採番にしているのは、付加情報は再生成可能（写真本体が真実の源）だから。軽微な追加列は SwiftData の軽量マイグレーションに任せる（例: 座標列の追加）。

Dropbox 側のモデル

CachedDropboxItem（path unique・contentHash で無効化）/ CacheUsageEntry（LRU 台帳）/ DropboxSyncState（カーソル）。詳細は キャッシュ実装詳細。
