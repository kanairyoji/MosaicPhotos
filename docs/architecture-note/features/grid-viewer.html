グリッドと全画面ビューア

数万枚を滑らかにスクロールするため、グリッドは SwiftUI の LazyVGrid ではなく UICollectionView を UIViewRepresentable で載せている。

なぜ UICollectionView なのか

設計判断 6 万件規模では LazyVGrid + ScrollViewReader の scrollTo(id) が不安定で、右端スクラバーでの大ジャンプが効かなかった。contentOffset を直接操作できる UICollectionView に置き換えて根本解決した。差分更新・プリフェッチ・セル再利用も UIKit の成熟した仕組みに乗る。

構成要素

型役割
PhotoCollectionViewUIViewRepresentable。UICollectionViewDiffableDataSource + CompositionalLayout。Coordinator がスクロール検知を担う。
GridThumbnailCellサムネイルセル。再利用時に前の画像要求をキャンセル。
GridSectionHeaderView日付などのセクションヘッダ。
GridScrubberView右端の高速スクラバー（contentOffset ベース）。
PhotoPageView / FullPhotoView / PhotoInfoPanel全画面ページング・ズーム・情報パネル。

レイアウトの要点

列数からセル幅を fractionalWidth(1/cols) + contentInsets で決める。repeatingSubitem:count: は意図せず横長になる不具合があったため、subitems:[item] を使う。

月グループの密度（最大密度パッキング）

表示は dense / 月 / 年 の3モード（ズーム段階で切替）。月グループでは、写真の少ない月が「ヘッダー＋半端な1行」を量産して疎になりやすい。これを photoGridSections（純関数・テスト対象）の貪欲パッキングで詰める：連続する月を「合計が coalesceBelow 枚（＝1行ぶん）に達するまで」蓄積して1セクションに区切り、複数月にまたがるときは範囲ラベル（2024-01 – 2024-03）にする。末尾に残った1行未満の月は直前セクションへ畳み込む。これで各セクションは最低1行ぶん埋まり、孤立した小さい月・半端行の量産を解消する。

設計判断 coalesceBelow は固定値ではなく実際の列数 × 行数設定。当初これを実列数にしていたが、スナップショット再構築を抑える性能最適化の副作用で固定値（4）へ劣化し疎に戻った回帰がある。grouping==.month の列数はズーム段階で固定のため、coalesce はスナップショットのシグネチャに含めても dense/year のピンチ（列数変更）では不変＝再構築しない（6.7万件で 0.5〜1s の再構築を繰り返さない配慮を維持）。密度（1セクションを閉じるまでの行数 1/3/5）は GridSettingsKeys.monthSectionRows でユーザー設定でき、設定 → General → Photo Grid から変更する。行数を増やすほど見出し（範囲ラベル）が減り粗く・密になる。

スクロールと背景処理の協調

スクロール中（ドラッグ・減速）は Coordinator が onScrubbingChange を発火し、背景の重い処理（CLIP 埋め込み）を一時停止させる。これにより操作中のカクつきを防ぐ。

sequenceDiagram participant U as ユーザー participant CV as UICollectionView participant Co as Coordinator participant T as PhotoTagger（背景） U->>CV: ドラッグ開始 CV->>Co: scrollViewWillBeginDragging Co->>T: isInteracting = true（譲る） U->>CV: 指を離して減速終了 CV->>Co: scrollViewDidEndDecelerating Co->>T: isInteracting = false（再開）

サムネイルのプリフェッチ

UICollectionViewDataSourcePrefetching で先のセルを前もって要求。端末写真は PHCachingImageManager の先読み窓（FIFO・直近 8 窓のみ保持し古い窓は stopCaching）でメモリを有界に保つ。

全状態に共通の下部バー

PhotoSourceContentView はグリッド・未接続・空・失敗のどの状態でも、最下部に Home / Settings バーを表示する。状態が変わってもナビゲーションを失わない。
