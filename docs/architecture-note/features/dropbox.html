Dropbox 連携

外部 SDK を使わず、URLSession と AuthenticationServices だけで OAuth 認証・差分同期・キャッシュを実装している。

認証（OAuth 2.0 + PKCE）

ASWebAuthenticationSession で標準フローを行う。カスタム URL スキーム MosaicPhotos://oauth/dropbox でコールバックを受ける。PKCE により、機密でないクライアントでも認可コード横取りを防ぐ。

sequenceDiagram participant App participant ASWeb as ASWebAuthenticationSession participant DBX as Dropbox App->>App: code_verifier 生成 → code_challenge App->>ASWeb: 認可URL（challenge付き） ASWeb->>DBX: ユーザー認可 DBX-->>App: 認可コード（カスタムURLで返却） App->>DBX: コード + code_verifier でトークン交換 DBX-->>App: access / refresh トークン App->>App: Keychain に保存

トークンは Security（Keychain）に保存。appKey と redirectURI はアプリ層の DropboxConfig（シークレットは gitignore 対象の DropboxSecrets）に置き、DropboxCore は設定値を持たない。

API 呼び出しの集約

認証ヘッダ付与とステータス検証を伴う RPC / コンテンツダウンロードは DropboxAPIClient（rpc / contentDownload）に集約する。longpoll など特殊なものだけ HTTPClient を直接使う。HTTPClient は抽象化されており、テストでスタブに差し替えられる。

差分同期

DropboxSyncEngine が list_folder のカーソルで差分を取り、longpoll で変更を待つ。カーソルは SwiftData（DropboxSyncState）に保持。初回フルスキャンはページごとに頻発するため、再フェッチ/再代入をスロットリング（先頭即時 + 以降は間引き）してメイン負荷を抑える。

graph LR LF[list_folder] --> CUR[(カーソル保存)] CUR --> LP[longpoll 待機] LP -->|変更あり| DELTA[差分取得] DELTA --> CACHE[(キャッシュ更新)] DELTA --> CUR

キャッシュ機構

DropboxCacheStore（actor）が、メタデータを SwiftData（CachedDropboxItem 等）、バイナリを ImageCacheKit で Caches/DropboxKit/{thumbnails,fullimages}/ にハッシュ名で保存する。contentHash 変更で無効化し、CacheUsageEntry（最終アクセス）ベースの LRU 容量管理を行う。

UI 層との分離

ロジックはすべて DropboxCore（SwiftUI 非依存）。UI（DropboxKit）は PhotoStore 適合・サムネセル・設定画面を担う。詳細な診断は DropboxDebugSection（Developer Options で合成）に分離。
