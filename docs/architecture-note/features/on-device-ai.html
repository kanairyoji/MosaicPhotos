オンデバイス AI

固定の語彙リストや OCR を持たず、任意言語の自然文で AI アルバム／意味検索を行う。実装は「タグ台帳＋LLM 審査」の多層構成（ADR-23/24）：シーンタグ・CLIP 埋め込み・VLM キャプションを夜間に付与し、検索は決定的ハード条件＋タグ一致＋CLIP 対比＋字句の RRF 融合＋LLM 審査で決める。通信なし・APIキー不要。

索引（夜間パイプライン）と検索

索引は夜間バッチ（電源＋Wi-Fi＋アプリ非使用の BGProcessingTask・実行タイミング参照）で、全写真へ次の順に付与する。

  1. Vision シーンタグ（OS 内蔵 VNClassifyImageRequest・約 1,300 クラス・hasMinimumRecall(forPrecision:) の校正済み足切り＝自前閾値が消える）
  2. CLIP 埋め込み（OpenCLIP ViT-B-32・意味検索の索引・fp16）
  3. VLM キャプション（SmolVLM-256M・写真ごとの英語短文・LLM 審査の「目」）

なぜ「閾値レス」へ再設計したか（ADR-24） 以前は生 CLIP コサイン＋絶対閾値だったが、ユーザーのライブラリ分布が事前に不明で閾値を調整できず「除外が 97% を落とす」「意味採用 0 件」が起きた。そこでタグ一致（離散・閾値レス）を第 3 のランキングに加え、除外は肯定/否定ベクトルの相対判定（絶対閾値なし）にし、最終判断をLLM 審査（言語判定）に委ねた。判定根拠がタグ/キャプションで可視化されデバッグ可能になる。
解釈は「作成時に 1 回だけ」（ADR-23） 検索文→QuerySpec・英訳の LLM 解釈はアルバム作成/編集時に 1 回だけ実行して AIAlbumInterpretationStore（JSONFileStore・版管理）へ永続化する。起動時・写真追加時に LLM は走らない（旧方式は毎起動で全アルバム再解釈し 9.4s のメインハングになっていた）。再評価は増分（新規埋め込み分だけ採点）＋ドリフト検知のフル再評価。

翻訳を前段に置く理由

CLIP（OpenCLIP ViT-B-32）のテキストエンコーダは英語中心。日本語クエリをそのまま入れると精度が落ちる。そこで Foundation Models（オンデバイス LLM）でクエリを英語へ正規化してから CLIP に渡す。翻訳もオンデバイスなので通信不要。翻訳失敗はキャッシュしない（誤訳の固定化を避ける）。

検索の融合（RRF）と証拠ゲート・審査

検索は 4 層。決定的なハード条件で絞り、複数のランキングを Reciprocal Rank Fusion で束ね、除外つきは証拠を要求し、最後に LLM が言語で判定する。

  • P0 接地: 日付は RelativeDateParser（日英）を唯一の出典・場所/人物はカタログ/原文出現に接地。QuerySpecSanitizer がプレースホルダ除去・include/exclude 衝突解消。
  • P1 融合: Vision シーンタグ一致（離散・閾値レス）＋CLIP 対比（除外は肯定/否定ベクトルの相対判定）＋LexicalSearch（地名/人物）を AIAlbumSearcher が RRF 融合。除外語はタグの離散一致でもハード除外。
  • P2 証拠ゲート＋審査: 除外つきはタグ/顔実測/キャプションの証拠を必須にし、上位候補を AlbumVerifier（Foundation Models）が keep/drop/unsure 判定。unsure は最大 2 回の再判定で多数決。空振り時はプローブ語を生成して 1 回だけ再検索（Refine）。

知覚 seam（プロトコルと実装の分離）

AutoAlbumCore はプロトコルだけを定義し、Core ML / Vision / Foundation Models 依存の実体は MobileCLIPKit が実装する。アプリの AutoAlbumAdapters が結線する。

ピープル（顔クラスタ）

写真アプリの「ピープル」は公開 PhotoKit API でアクセスできない（旧 subtype-1000 方式は常に空だったため撤去）。代わりに Vision 顔検出（VNDetectFaceRectanglesRequest）＋同梱顔モデル（facenet・512 次元 L2 正規化）で identity 埋め込み→逐次クラスタリングで自前の「人物」を作る。ロジックは AutoAlbumCore/Faces（FaceClustering 純・FaceStore は別コンテナ "FacesV1"＝CLIP データを壊さない・PeopleEngine ファサード）。端末写真のみ対象で、顔モデル未同梱ならセクション非表示。

表示タグは検索と同じ台帳

フル画像に出すキーワードタグは Vision シーンタグ（TagStore・検索の一次ランキングと同一ソース）を第一に、CLIPDisplayLabeler（約 300 語ゼロショット・保存済み埋め込み使用）で補完する。VLM キャプションも「AI description」欄に表示（生成済みのみ）。タグ/キャプションは TagsV1 別コンテナで、夜間バッチ（TagTagger）が付与する。

背景処理のスロットリング

シーンタグ・CLIP 埋め込み・顔スキャンは BackgroundTrickle の骨格で小バッチ＋バッチ間スリープ＋.background QoS で trickle 実行する。重さは設定（段階）で調整でき、停止判定は 1 枚単位（ADR-25）。ユーザー操作中・メモリ圧迫中・クラウド取得中は処理を譲る。シミュレータでは背景推論を実行しない（CPU で数秒かかり検証を妨げるため）。詳細は バックグラウンドと省電力。
