バックグラウンドと省電力・通信制御

シーンタグ付け・CLIP 埋め込み・VLM キャプション・顔スキャン・自動アルバム生成といった重い処理は、原則としてアプリを使っていない時間帯（画面ロック中の BGProcessingTask）にだけ動かし、フォアグラウンドでは一切走らせない（ADR-25）。実行タイミングはユーザーが 5 段階で選べる。状況は最上部のアクティビティバーで可視化する。

なぜ「使っていない時だけ」なのか（ADR-25） 旧ゲートは「電源＋低電力OFF＋アイドル 60 秒」だった。充電しながら閲覧中に 60 秒触らないと重い処理が走り出し、操作を再開してもクラウド写真のタグ付け（1 バッチ最大約 27 秒）が譲れず操作が固まった。そこでアイドル判定を廃止し、「電源＋低電力OFF＋アプリ非アクティブ（scenePhase）＋Wi-Fi」に変更。フォアグラウンドでは重い処理をゼロにした。停止判定も全処理1 枚単位に統一し、操作・遷移が来たら即譲る。

重い処理の実行タイミング（5 段階設定 / HeavyWorkTiming）

固定ゲートをやめ、実行タイミングをユーザーが 5 段階で選択できる（HeavyWorkTiming・純関数 allows(...) で判定）。段階は単調に条件を緩める。低電力モード・メモリ圧迫はどの段階でも常にブロックする安全弁。

段階いつ重い処理を動かすか
一時停止 (paused)手動「今すぐ処理」以外は動かさない
おまかせ (nightly・既定)電源＋Wi-Fi＋アプリ非使用（画面ロック/切替中）。実質は夜間の BGProcessingTask
充電中は合間も (chargeActive)上記に加え、充電中はアプリ使用中の合間も（全タッチを TouchActivityTracker で捕捉し 20 秒アイドルで開始・タッチで即停止）
バッテリーでも (battery)電源につないでいなくても残量 20% 以上なら実行（BGTask の requiresExternalPower も外す）
制限なし (unlimited)モバイル回線も含めいつでも

中央ゲートは BackgroundYield.heavyWorkAllowed。BGTask 起動時は scenePhase 変化が来ないため runHeavyWork 冒頭で非アクティブを明示する（初期値のままゲートが開かない罠への対処）。設定 UI は Albums の Auto Albums 側「処理のタイミング」に置き、各段の条件と影響を footer に明記する。

AI アルバム作成は 2 段階（即時プレビュー→夜間本番化）

重い処理を使用中に走らせないため、AI アルバムの作成は 2 段階に分ける。作成時は決定的プレビュー（レキシコン＋日付＋タグ照合・LLM なし・1〜2 秒）で即座にメンバーを見せ、pendingFinalization を立てる。実際の LLM 解釈・証拠ゲート・審査（finalizePending）は夜間パイプラインで本番化する。作成画面にはこのタイミングを明記する。

電源ゲート（PowerStateMonitor）

UIDevice.batteryState と ProcessInfo.isLowPowerModeEnabled を監視し、ポリシー BackgroundPowerPolicy（While charging=既定 / Always / Off）と合わせて backgroundAllowed() を返す。While charging は「電源接続（charging/full）かつ 低電力モード OFF」。

シミュレータ注意 シミュレータは充電状態を .unknown で返すため、charging/full のみで判定すると背景処理が全部止まる。「.unplugged と確定したとき以外は電源扱い」にしてロックを避ける。

回線ゲート（NetworkStateMonitor）

NWPathMonitor で isOnWiFi（Wi-Fi/有線）・isExpensive（従量）・isConstrained（低データモード）を監視し、ポリシー BackgroundDataPolicy（Cellular allowed / Wi-Fi only=既定 / Wi-Fi skip Low Data / Off）と合わせて networkAllowed() を返す。通信を伴う背景処理は backgroundAllowed() かつ networkAllowed() で実行する。

CLIP 埋め込みのスマート方針

埋め込みは回線NG時にクラウド写真（サムネDL）をスキップしてローカル写真だけ続行する（unembeddedRefKeys(limit:localOnly:) で DB 側に "L-" 前缀で絞り込み）。ローカルが尽きたら終了し、Wi-Fi/電源の復帰（onChange）で scheduleBackgroundFill() を再起動してクラウド分を拾い直す。

可視化：アクティビティバー

画面最上部の 1 行に、左から電源チップ（稼働可=緑⚡／電池待ち=橙／Off=灰）・回線チップ（Wi-Fi=緑／保留=橙／Off・圏外=灰）・Dropbox 通信（サムネスロット・同期・DL・アップロード）・背景処理（AI 埋め込み＝残り枚数併記・アルバム生成・場所走査・アルバム走査、稼働中はパルス）を並べる。形＝チャンネル／色＝状態／数・塗り＝強度。各処理は BackgroundActivityMonitor / DropboxActivityMonitor / 各モニタへ報告し、バーが @Observable として購読する。設定「Show activity bar」（既定 ON）で表示。

設定

アプリ横断の設定なので、機能別ではなく設定ルートの General → Background & Battery（BackgroundSettingsView）に「Power」「Background Data」を並べて置く。背景 CLIP の速度段階は Albums の Auto Albums 側（処理の重さ）に残す。
