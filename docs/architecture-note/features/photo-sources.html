写真ソースの統一

端末写真・Dropbox・両者の統合を、同じプロトコルと同じビューで扱う。新しいソースを足しても表示側は変わらない。

中心となるプロトコル

PhotoSourceKit が表示インターフェイスを定義する。

型役割
PhotoItem1 枚の写真の抽象（id・撮影日など）。ローカル/クラウドで実体が異なる。
PhotoLoadingアイテム単位の取得（thumbnail / fullImage / prefetch / location / metadata）。
PhotoStore: PhotoLoading一覧と状態（state: PhotoLoadState）を持つストア。
PhotoLoadState権限要求 / 読み込み中 / 完了 / 空 / 失敗 を表す enum。
PhotoInsightフル画像の付加情報（表示タグ・人物・解析状態）。SwiftUI 非依存値型。

3 つの実装

graph TD PS["PhotoStore（プロトコル）"] LPS[LocalPhotoStore] --> PS DPS[DropboxPhotoStore] --> PS MPS[MergedPhotoStore] --> PS MPS --> LPS MPS --> DPS
  • LocalPhotoStore（LocalPhotoCore）— PhotosKit で PHAsset を扱う。
  • DropboxPhotoStore（DropboxCore）— Dropbox API とキャッシュ。
  • MergedPhotoStore（PhotosFeatureKit）— 上記 2 つを統合。MergedPhotoItem は enum で、id にプレフィックス（L-/C-）を付けて衝突を避ける。

統合時の状態解決

MergedPhotoStore.resolveState は純関数で、ローカルの状態とアイテム有無から全体状態を決める。たとえばローカルが空でも Dropbox が取得中なら .empty ではなく .loading を返す（「写真なし」を早まって出さない）。純関数なのでユニットテストしやすい。

// アイテム無し時はローカルの状況に従うが、Dropbox 取得中は loading を維持
nonisolated static func resolveState(localState:hasLocalAssets:hasDropbox:dropboxBusy:) -> PhotoLoadState

付加情報の受け渡し（レイヤ分離）

フル画像ビューはタグや人物を出すが、PhotoSourceKit は AI 層（AutoAlbumCore）に依存しない。そこで付加情報は photoInsight という環境クロージャとして注入する。実体は AutoAlbumEngine.insight。

設計判断 表示側を 1 つのプロトコルに統一することで、グリッド・全画面・スクラバー・状態バーといった UI を全ソースで共有できる。逆に AI への依存を環境クロージャで切ることで、基盤パッケージが上位レイヤを知らずに済む。
