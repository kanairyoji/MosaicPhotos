場所アルバム

ローカルと Dropbox の位置情報を市区町村ごとにまとめる。逆ジオコーディングは重いので、グリッド集約とキャッシュで呼び出しを最小化する。

座標の収集

ローカルは PHAsset.location を優先し、無ければ EXIF の GPS を直接読み取って補完する（読み取り結果は localIdentifier 単位でキャッシュ）。Dropbox は同期済み media_info の座標を使う。

グリッド集約 → 逆ジオコーディング

全座標をそのまま逆ジオコーディングすると CLGeocoder の呼び出しが膨大になる。そこで座標を粗いグリッド（GeoGridKey）にまとめ、ユニークなグリッドキーごとに 1 回だけ地名解決する。解決結果も永続キャッシュする。

graph TD L[ローカル座標
PHAsset / EXIF] --> M[候補集合] C[Dropbox 座標
media_info] --> M M --> G[GeoGridKey で粗くグリッド化] G --> U[ユニークキーごとに逆ジオコーディング] U --> R[PlaceNameResolver
actor・地名キャッシュ] R --> CITY[市区町村ごとに集約] CITY --> ALB[Places アルバム]

オフメインと段階表示

67k 件規模の compactMap・グリッド集約・署名計算は Task.detached(priority: .utility) でオフメイン実行する（メインで回すと起動・定期スキャンでカクつく）。初回は地名解決の都度 publish して段階表示し、再スキャンは無瞬断のため最後に一括反映する。

差分での再スキャン

Dropbox の座標集合の署名（並び順非依存の XOR ハッシュ）と、PHPhotoLibraryChangeObserver による端末変更フラグで、変化があるときだけ再スキャンする。バックグラウンド同期で座標が補完されると Places アルバムが動的に増える。

設計判断 逆ジオコーディング（CLGeocoder）はネットワーク・レート制限・遅延がある重い操作。グリッド集約でユニーク化 + 永続キャッシュすることで呼び出し回数を桁違いに減らし、再スキャンを安価にしている。
