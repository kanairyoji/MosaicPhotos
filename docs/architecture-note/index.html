MosaicPhotos 設計資料

端末内と Dropbox の写真を統一的に閲覧し、語彙に縛られない自然文検索をオンデバイス AI で実現する iOS アプリ。その構造・設計判断・使っている技術をまとめた内部資料です。

この資料の二つの目的 1. 構造の把握 — どの層がどう責務を持ち、なぜそう分けたか。
2. 技術の学習 — Swift Observation / SwiftData / PhotosKit / CLIP / Core ML / OAuth など、本アプリが用いる技術そのものを理解する。

このアプリを一言で

外部 SDK を使わず標準フレームワークのみで構成。写真ソース（端末 / Dropbox）を共通インターフェイスに統一し、「タグ台帳＋LLM 審査」の多層構成（ADR-23/24）で語彙に縛られない自然文検索を行う。夜間に Vision シーンタグ → OpenCLIP 埋め込み（Core ML）→ VLM キャプションを全写真へ付与し、検索は決定的ハード条件＋タグ一致＋CLIP 対比＋字句を RRF 融合し、Foundation Models の LLM 審査で決める。通信なし・APIキー不要。
