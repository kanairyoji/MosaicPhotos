# 操作性（UX）監査 — 所見リスト

> アプリ全体の**操作性**を主要フローに沿って点検した所見の正本。`decisions.md`（ADR）/ `case-studies.md`（事例）とは別の**レビュー成果物**で、ここでは「動線・摩擦・発見性・一貫性・フィードバック・アクセシビリティ」を見る。
>
> **運用ルール**
> - 1 件 = `所見 / 影響 / 提案 / 優先度 / 関連`。優先度は **P1（機能的な穴・最頻操作）> P2（摩擦・到達性・発見性）> P3（磨き・設計判断）**。
> - 着手したら各項の末尾に「状態（着手/実装/見送り＋コミット）」を追記して経緯を残す（項目は消さない）。
> - 設計判断を伴うもの（ホーム構造の変更等）は ADR にも起こす。
>
> 監査日: 2026-06-29 / 対象ブランチ: main（`3b04e23a` 時点）。実機未検証の項は「要実機」と明記。

## 動線（ナビゲーション）

### N1. フル画面の戻る/閉じるが左上ボタンのみ — **P1**
- 所見: 先日ナビバーを `toolbar(.hidden)` にしてカスタム戻るボタン化した副作用で、**横エッジスワイプ戻る**が無効。さらに写真ビューア定番の**下スワイプで閉じる**も未実装。閉じる手段は左上ボタン1つだけ。
- 影響: 片手操作で一番触る操作。親指の届きにくい左上だけ、というのは摩擦が大きい。
- 提案: フル画面に **下スワイプで閉じる**（最上部での overscroll を検出して dismiss）を追加。注意点＝`FullPhotoView` は縦 `ScrollView` で写真の下に情報パネルを置く構造なので、スクロール位置 0 からの下方向ドラッグだけを dismiss に割り当て、情報パネルのスクロールと競合させない。エッジ戻るは戻すなら別途。
- 関連: `PhotoPageView.swift`（topControls/dismiss）/ `FullPhotoView.swift`（縦 ScrollView）。
- 状態: 未着手。

### N2. ホームが縦長の単一リスト（7 セクション） — P3
- 所見: Sources / Time&Place / People / AI / Folder / Device Albums / Places が 1 本の `List`。ソース（写真/クラウド/すべて）の切替は毎回ホームへ戻る前提で、常時アクセスできるタブが無い。
- 影響: 閲覧中に別ソースへ移るのに「閉じる→スクロール→選ぶ」の手数。ヘビーユースで効く。
- 提案（設計判断・要 ADR）: (a) 主要 3 ソースを下部タブ化、(b) ホーム最上部にセグメント、(c) 現状維持＋ソースビュー下部バーにソース切替を追加、のいずれか。まずは小さく (c) を検討。
- 関連: `HomeView.swift` / `Home/HomeSections.swift` / `PhotoSourceContentView`（bottomBar）。
- 状態: 未着手（要 ADR）。

## 機能の穴

### F1. お気に入りが「表示専用」 — **P1**
- 所見: グリッド左下・フル画面右上にハートを出すようにしたが、**アプリ内で付け外しできない**（読み取りのみ）。
- 影響: 「見えるのに操作できない」のは写真ビューアとして片手落ち。ハートを見たユーザーは押せると期待する。
- 提案: フル画面のハートを**タップでトグル**（`PHAsset` の `isFavorite` を `PHPhotoLibrary.performChanges` で更新）。端末写真のみ対象、クラウドは非表示のまま。トグル後はグリッド/ライブラリ変更通知で反映。ハートのダブルタップ追加も定番。
- 関連: `PhotoPageView.swift`（heart）/ `LocalPhotoStore`（PhotoKit 書き込み seam を追加）/ `PhotoLoading`（任意で `setFavorite` を追加）。
- 状態: 未着手。

### F2. 写真権限の `needsSetup` に「設定を開く」ボタンが無い — P2
- 所見: 権限未許可時の `setupView` は「Please allow access in the Settings app.」を**文言表示するだけ**。`LocalPhotoStore+PhotoStore` に `openSettings` ロジックはあるのにプレースホルダのボタンに繋がっていない。
- 影響: ユーザーが手動で設定アプリへ行く必要があり、復帰率が下がる。
- 提案: `PhotoLoadState.needsSetup` に任意のアクション（例 `openSettings`）を持たせ、`setupView` に「設定を開く」ボタンを出す。`UIApplication.openSettingsURLString` を開く。
- 関連: `PhotoLoadState.swift` / `PhotoSourceContentView.setupView` / `LocalPhotoStore+PhotoStore`。
- 状態: 未着手。

## 発見性

### D1. 隠れジェスチャにヒント/初回ガイドが無い — P2
- 所見: グリッドの**ピンチで密度変更**、フル画面の**情報パネル（下スクロール）**、**スクラバー**は、初見では存在に気づけない。初回ガイドやヒントが無い。
- 影響: 主要機能が「知っている人だけ」のものになる。
- 提案: 軽量に＝(a) 初回だけ短いコーチマーク（"ピンチで表示サイズ変更" 等・1 回表示して `@AppStorage` で抑制）、(b) ヘルプに既に基本ジェスチャ章があるなら相互リンク。重い tutorial は不要。
- 関連: `PhotoGridView`（pinch）/ `FullPhotoView`（info panel）/ `GridScrubberView` / `docs/help/basics.html`。
- 状態: 未着手。

### D2. 情報パネルのアフォーダンスが無い — P2
- 所見: 写真の下に情報パネルがあるが、スクロールインジケータ非表示で「下に何かある」手掛かりが無い。
- 提案: 写真下端に控えめなシェブロン（`chevron.compact.up` 等）や「i」バッジを薄く出し、上スワイプで開くことを示す。
- 関連: `FullPhotoView.swift`。
- 状態: 未着手。

## アクセシビリティ

### A1. accessibility 指定が全体で 4 箇所のみ — P2
- 所見: `accessibilityLabel/Hint/Element` の使用が全コードで 4 件。グリッドセル（写真）・カルーセル・ハート・各アイコンボタンが VoiceOver で意味を持たない可能性。Dynamic Type（特に固定 `font(.system(size:))` 多用箇所）・ヒット領域（34pt は可、非interactiveハートは可）も未検証。
- 影響: VoiceOver 利用者に厳しい。App Store のアクセシビリティ観点でも弱い。
- 提案: (a) グリッドセルに「写真・日付・お気に入り」のラベル、(b) 各アイコンボタンへ `accessibilityLabel`、(c) アクティビティバーは装飾として `accessibilityHidden(true)`、(d) 主要文字の Dynamic Type 追従を実機 VoiceOver で点検（要実機）。
- 関連: 横断（`GridThumbnailCell` / `PhotoPageView` / `HomeRows`（PeopleCarousel 等））。
- 状態: 未着手（要実機検証）。

## 空状態・初回

### E1. 空状態・未接続からの「次の一手」が弱い — P2
- 所見: `emptyView` は「No photos yet.」のみ。Dropbox 未接続のクラウドビューや、AI/フォルダ/場所アルバムの 0 件状態で、次に何をすればよいかの導線が弱い（一部は設定への文言はある）。
- 提案: 空状態に**行動ボタン**を添える（例: クラウド空→「Dropbox に接続」、AI 0 件→「アルバムを作成」、写真空→「設定を開く」）。F2 と方針を共通化。
- 関連: `PhotoSourceContentView.emptyView` / `DropboxContentView` / `HomeSections`（各セクションの空状態）。
- 状態: 未着手。

## 一貫性・磨き

### C1. ハート表現の不一致 — P3
- 所見: グリッド=左下・白＋影、フル画面=右上・ピンク＋材質丸。位置も色も別。
- 提案: 色・形を片方に寄せる（例: 両方ピンクの塗りハート、もしくは両方白）。配置はそれぞれの画面に合わせてよいが、色/塗りは統一。
- 関連: `GridThumbnailCell` / `PhotoPageView`。
- 状態: 未着手。

## 良かった点（維持したい）
- ソース横断で `PhotoSourceContentView`→グリッド→詳細が**一貫**。全状態の下部に Home/Settings バーがあり迷子になりにくい。
- 失敗状態に **Retry** ボタンがある（`failedView`）。
- 設定が Settings.app 風グルーピングで馴染みがある。
- 直近の遷移高速化（6.7万件 TabView → ウィンドウ化）で**写真を開く操作が 14s→0.2s**。フル画像ロード中のサムネ先出しも実装済み。
