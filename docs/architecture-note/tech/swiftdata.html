SwiftData

Core Data を Swift らしく包んだ永続化フレームワーク。クラスに @Model を付けるとスキーマになり、ModelContext で fetch / insert / delete / save する。

基本要素

要素役割
@Model永続化対象のクラス。プロパティが列になる。@Attribute(.unique) で一意制約。
ModelContainerスキーマとストア（ファイル/インメモリ）の入れ物。
ModelContext変更の作業領域。スレッド安全でない。
FetchDescriptor取得条件（#Predicate・並び・fetchLimit/fetchOffset）。
@ModelActorコンテキストを専用アクタに束縛し、安全に並行アクセスする。

このアプリでの落とし穴と対策

1. @Model を漏らさない @Model は参照型でアクタ非分離。アクタ外へ渡すと競合する。本アプリは @ModelActor の外へ @Model を出さず、Sendable な値型（EnrichedPhoto 等）に変換して返す。
2. fetch は属性を丸ごと展開する 大きな Data を @Model に inline で持つと、全件 fetch のたびに blob も展開してメモリを食う。本アプリは CLIP 埋め込みを PhotoEmbedding 別テーブルに分け、メタデータ fetch が blob に触れないようにした（→ 事例）。
3. 起動時 trap を避ける 破損・スキーマ不整合で ModelContainer 初期化が trap する。本アプリは makeResilientContainer で「削除→再試行→インメモリ」とフォールバックする（→ 診断と堅牢性）。スキーマ変更時は ModelConfiguration 名（例 "AutoAlbumV10"）を採番して旧ストアを破棄する。

メモリを有界に保つ書き込み

大量 upsert は writeChunk 件ごとに使い捨ての ModelContext を作って save し、チャンク完了でその context を解放することで登録オブジェクトを溜めない。
