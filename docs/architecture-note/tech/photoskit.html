PhotosKit

端末の写真ライブラリにアクセスする Apple のフレームワーク。アセットの列挙・画像取得・変更監視を担う。

主要な型

型役割
PHPhotoLibrary権限要求・ライブラリ全体・変更監視。
PHAsset写真 1 枚のメタデータ（撮影日・座標・サイズ・お気に入り等）。実体ではなく軽量な参照。
PHFetchResultアセットの遅延コレクション。enumerateObjects で走査。
PHImageManager / PHCachingImageManagerサムネ/フル画像の非同期取得と先読み。
PHAssetCollectionアルバム・スマートアルバム（People など）。

権限

初回アクセス時に requestAuthorization(for: .readWrite) で許可を求める。Info.plist の NSPhotoLibraryUsageDescription が必須（本アプリは pbxproj の INFOPLIST_KEY_* で設定）。限定アクセス（.limited）もある。

気をつける点

  • コールバックはバックグラウンド — PHImageManager の completion は別スレッドで来るので Task { @MainActor in } でメインへ戻す。
  • 全件配列化はメモリに注意 — enumerateObjects で [PHAsset] 化するのは O(枚数)。本体画像ではなく参照なので軽いが、数万件では無視できない。
  • EXIF を読むときは本体データを取る — PHAsset.location が無い場合の GPS 補完は requestImageDataAndOrientation で本体を読む（重いのでキャッシュ前提）。

このアプリでの使われ方

  • LocalPhotoStore — 一覧・権限・サムネ取得・先読み窓（FIFO）。
  • LocalAlbumScanner — PHAssetCollection 走査（バックアップと独立）。
  • PlaceScanner — 座標収集（PHAsset.location → EXIF 補完）。
  • BackupAssetReader — バックアップ用の本体データ取得。
