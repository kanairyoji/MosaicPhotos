SPM モジュール分割

Swift Package Manager のローカルパッケージで、機能を独立したモジュールに分ける。境界が明確になり、ビルド・テストが速くなる。

ローカルパッケージとは

リポジトリ内の Packages/Foo/Package.swift をアプリから参照する。各パッケージは独立した target と tests を持ち、依存を .package(path:) で宣言する。

分割の効能

  • 一方向依存の強制 — UI がロジックに依存し、逆はできない（コンパイラが保証）。
  • 高速テスト — 純ロジックは macOS の swift test で UI を起動せず回る。
  • 関心の分離 — 重い依存（Core ML）を 1 パッケージに隔離できる。

このアプリの工夫

手法狙い
Core/UI 分離（DropboxCore / DropboxKit）SwiftUI 非依存のロジックを独立して進化・テスト
@_exported importUI パッケージが Core を再エクスポート。利用側は 1 import で済む
seam をプロトコル化（AutoAlbumCore）＋実装を別パッケージ（MobileCLIPKit）Core ML 抜きで AI ロジックをテスト
各パッケージで .iOS(.v17) 宣言 + @available ゲートアプリは iOS 26 でも、パッケージは macOS テストを維持
既知の落とし穴 あるパッケージに新規ファイルを追加すると、それに依存する別パッケージの .build が stale になり「型が見つからない」で swift test が落ちることがある。rm -rf Packages/*/.build してから再実行で解消する。

アプリからの参照

新しいパッケージをアプリが直接参照する場合は project.pbxproj に XCLocalSwiftPackageReference / XCSwiftPackageProductDependency / Frameworks フェーズの配線が要る。既存パッケージに transitive 依存するだけの新パッケージはアプリが直接参照しないため pbxproj 変更不要。
