Core ML / Foundation Models

Core ML はオンデバイス推論の基盤（CLIP を動かす）。Foundation Models は OS 内蔵の小型 LLM（クエリ翻訳に使う）。どちらも通信不要。

Core ML

学習済みモデルを .mlpackage → コンパイルして .mlmodelc で同梱し、MLModel で推論する。入出力は MLFeatureValue（画像なら MLImageConstraint、テキストなら MLMultiArray）。

let model = try MLModel(contentsOf: url, configuration: config)
let provider = try MLDictionaryFeatureProvider(dictionary: [inputName: featureValue])
let out = try model.prediction(from: provider)
let vec = out.featureValue(for: outputName)?.multiArrayValue   // 512次元
このアプリでのハマりどころ
  • 画像エンコーダの変換は compute_precision=FLOAT16。実機の ANE は fp16 前提で、fp32 だと ANE 非対応＝埋め込みが大幅に遅くなる。fp16 は一部シミュレータで NaN 化し得るが、ランタイムの有限性チェックが nil に落とす（画像タワーの検証は実機）。
  • 前処理に mean/std 正規化を入れない（MobileCLIP は Resize/CenterCrop/ToTensor のみ）。
  • 実行ユニットはシミュレータ .cpuOnly・実機 .all（シミュレータは ANE/MPSGraph が無く .all だと Espresso 例外）。
  • モデルは遅延ロード（static let shared）で起動を重くしない。ロード結果は診断ログへ。

Foundation Models

Apple Intelligence のオンデバイス基盤モデル（import FoundationModels）。本アプリでは検索クエリの英語正規化（翻訳）に使う。CLIP のテキスト側が英語前提のため、前段で言語を揃えると精度が上がる。通信なし・APIキー不要。

graph LR Q["日本語クエリ '紅葉の写真'"] --> FM[Foundation Models] FM --> EN["'autumn leaves'"] EN --> CLIP[MobileCLIP テキスト埋め込み]

同梱の扱い

CLIP モデルと語彙は MosaicPhotos/MobileCLIP/ に置き、サイズの都合で .gitignore 対象（scripts/build_mobileclip.sh で生成）。未同梱でもアプリは動作し、CLIP 機能だけ無効化される。MobileCLIP.modelsBundled でロードせず同梱判定できる。
