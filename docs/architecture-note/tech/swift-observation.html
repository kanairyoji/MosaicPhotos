Swift Observation（@Observable）

iOS 17 で導入された状態観測の仕組み。@Observable を付けた型のプロパティを SwiftUI が自動追跡し、参照したプロパティが変わったときだけ再描画する。

基本

@Observable
final class LocalPhotoStore {
    var assets: [PHAsset] = []   // 参照しているビューだけが再描画される
    var state: PhotoLoadState = .idle
}

ビュー側は @State で保持するか、関数引数で受け取るだけ。@Published も ObservableObject も不要。

@ObservableObject との違い

観点ObservableObject(Combine)Observation
追跡の粒度オブジェクト単位（objectWillChange）プロパティ単位（参照したものだけ）
宣言@Published を各プロパティに@Observable を型に 1 つ
再描画過剰になりやすい必要最小限
このアプリでの方針 本アプリは @Observable に統一し、ObservableObject/Combine は使わない（iOS 26 SDK では import Combine が明示的に必要になるなどの摩擦を避ける）。@ObservationIgnored を使えば、観測対象から外したい内部依存（ロガー・キャッシュ・seam）を保持できる。

このアプリでの使われ方

  • LocalPhotoStore / DropboxPhotoStore / MergedPhotoStore — 一覧と状態を @Observable で公開。
  • AutoAlbumEngine / BackupEngine / PlaceScanner — @MainActor @Observable のファサード。
