CLIP と埋め込み

CLIP は画像とテキストを同じベクトル空間に写す。「写真のベクトル」と「文のベクトル」の近さ（コサイン類似）を測れば、語彙リストなしで自然文検索ができる。

埋め込み（embedding）とは

画像や文を、意味を表す固定長の数値ベクトル（本アプリでは 512 次元）に変換したもの。似た意味のものはベクトル空間で近くに配置される。

graph LR IMG[画像] --> IE[画像エンコーダ] --> VI["ベクトル(512次元)"] TXT["文 'a red umbrella'"] --> TE[テキストエンコーダ] --> VT["ベクトル(512次元)"] VI --- COS{コサイン類似} VT --- COS COS --> SCORE["近さ = 一致度"]

コサイン類似

2 ベクトルのなす角の cos。1 に近いほど同方向＝意味が近い。長さに依存しないので埋め込み比較に向く。

cos(a, b) = (a・b) / (|a| × |b|)   // -1 … 1

ゼロショット分類

学習し直さずに、候補ラベル群（例: "beach", "mountain", "food" …）をテキスト埋め込みにして、画像ベクトルと最も近いものを選ぶ手法。本アプリの表示タグはこの方式（約 300 語に対するゼロショット）。

オープン語彙検索

固定タグ方式は「決めた語」しか検索できない。CLIP は任意の文を埋め込めるので、語彙外の自由なクエリでも照合できる。これがオープン語彙。

このアプリでの実装メモ
  • モデルは OpenCLIP ViT-B-32 / DataComp（MIT）を Core ML 化（512 次元。ファイル名 MobileCLIP* は互換のため据え置き）。
  • 埋め込みは ClipMath で Data 化して永続化。検索用は fp32、保存は Float16（半分のサイズ）。
  • NaN/Inf を含むベクトルは壊れているので破棄（fp16 化の前処理で全滅を防ぐ）。
  • テキスト側は英語前提なので、クエリは Foundation Models で英訳してから埋め込む。
