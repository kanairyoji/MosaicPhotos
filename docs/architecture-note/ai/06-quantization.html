6. 量子化と精度

数値を粗く表すほどメモリと計算は軽くなるが、精度は落ちる。どこまで粗くしてよいかが量子化の勘所。埋め込み検索は意外と粗さに強い。

浮動小数点の精度

型ビット/要素1件512次元特徴
fp32（単精度）322048 B標準。精度十分・サイズ大
fp16（半精度）161024 Bサイズ半分。表現範囲は狭い
bf16161024 B範囲は fp32 並み・仮数は粗い（学習向き）
int88512 B1/4。スケール係数とセットで使う

スカラー量子化（int8 など）

実数値を、ある範囲に対する整数に丸める。復元時はスケール（と必要ならゼロ点）を掛けて戻す。1 要素 1 バイトまで縮むが、丸め誤差が乗る。

q = round((x - min) / scale)      // 量子化
x ≈ q * scale + min                // 復元（近似）

直積量子化（PQ）

ベクトルを小区間に分け、各区間を「代表ベクトルの番号」に置き換える。大幅に圧縮でき、距離計算もテーブル参照で高速化できる。ANN（第5章）と組み合わせる。

なぜ埋め込みは粗さに強いのか

検索で本当に必要なのは「どれが一番近いか」という相対順位であり、距離の絶対値ではない。多少の丸め誤差があっても順位は保たれやすい。とくに fp16 は、コサイン類似の値がほとんど変わらない。

graph LR V32["fp32 ベクトル"] --> Q[fp16/int8 に量子化] Q --> V16["軽いベクトル"] V16 --> COS[コサイン類似] COS --> R["順位はほぼ不変"]
実務の指針 まず fp16 でサイズ半減（精度ロスはほぼ無視できる）。さらに削るなら int8（順位はおおむね維持、わずかに recall 低下）。極端な圧縮（バイナリ化）は「粗いふるい→上位だけ高精度で再ランク」という二段構えで使う。

注意点

  • fp16 は表現範囲が狭い。極端に大きい/小さい値や NaN/Inf の混入に注意（壊れたベクトルは除外する）。
  • 保存は粗く、計算時に fp32 へ戻すのも一手（保存サイズだけ削る）。
  • 「モデルの重みの量子化」と「埋め込みの量子化」は別物。前者は推論速度・モデルサイズ、後者は索引サイズに効く。
