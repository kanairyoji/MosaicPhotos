3. Transformer と注意機構

いまの言語モデルも画像モデル（ViT）も、中心は Transformer。鍵は「系列の各要素が、関連する他の要素に注意（attention）を向ける」仕組み。

まずトークン化

文はそのままでは扱えないので、トークン（単語や部分文字列）に区切り、各トークンを埋め込みベクトルにする。画像の場合（ViT）は、画像を小さなパッチに分割し、各パッチをベクトル化する。

graph LR T["'a red car'"] --> TK["['a','red','car']"] TK --> EMB["各トークンの埋め込み列"] IMG[画像] --> PATCH["パッチ分割"] PATCH --> PEMB["各パッチの埋め込み列"]

self-attention（自己注意）

各トークンは「自分はどのトークンを参照すべきか」を計算する。たとえば "it" が文中の何を指すかを、関連の強い語に重みを置いて集約する。これにより文脈に応じた表現が得られる。

仕組みを乱暴に言うと、各要素から Query / Key / Value の 3 つのベクトルを作り、Query と各 Key の一致度（内積）で重みを決め、その重みで Value を加重平均する。

graph TD Q[Query] --> S[各 Key との一致度] K[Key] --> S S --> W[重み（softmax）] V[Value] --> O[加重平均 = 出力] W --> O

Transformer ブロック

self-attention の後に、各要素を独立に変換する小さな全結合層（FFN）を重ね、これを 1 ブロックとして何層も積む。multi-head（複数の注意を並列に持つ）で、異なる種類の関係を同時に捉える。

なぜ強いのか

  • 長距離の依存を 1 ステップで捉えられる（RNN のように逐次でない）。
  • 並列計算しやすく、大規模化と相性が良い。
  • テキストも画像も同じ枠組みで扱える（→ CLIP のように両者を組み合わせやすい）。

ViT（Vision Transformer）

画像をパッチ列として Transformer に入れる方式。CNN に代わり、CLIP の画像エンコーダにも使われる。出力を集約して 1 本の画像埋め込みにする。

この章の要点 Transformer は「要素同士の関連に注意を向けて文脈表現を作る」装置。テキストと画像を共通の方法で埋め込めることが、次章の CLIP につながる。
