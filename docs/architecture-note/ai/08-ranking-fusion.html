8. ランキングと融合

良い検索は単一の手がかりに頼らない。意味（ベクトル）・字句（語の一致）・条件（日付/場所）を別々に並べ、順位を融合して最終結果にする。

複数の手がかり

シグナル得意苦手
意味検索（ベクトル）言い換え・抽象的な内容固有名詞の厳密一致
字句検索（キーワード）固有名詞・正確な語言い換え・概念
構造化条件日付・場所・人物の絞り込み内容の意味

これらは互いの弱点を補う。だから併用して融合すると精度が上がる。

スコア融合の難しさ

各シグナルのスコアは尺度がバラバラ（コサインは -1〜1、BM25 は青天井）。素朴に足すと尺度の大きい方に支配される。これを避けるのが順位ベースの融合。

Reciprocal Rank Fusion（RRF）

各リストでの順位だけを使う。あるドキュメントのスコアを、各リストの順位 r の逆数で合計する。

RRF(d) = Σ  1 / (k + rank_i(d))      // k は定数（例 60）
         i（各リスト）

上位ほど寄与が大きく、複数リストで上位に出るものが勝つ。スコアの尺度差に影響されず、実装も単純で頑健。

graph TD A["意味検索の順位"] --> RRF[RRF 合算] B["字句検索の順位"] --> RRF C["条件フィルタの順位"] --> RRF RRF --> FINAL[最終ランキング]

再ランク（reranking）

「速いが粗い検索で候補を絞り、上位だけを重いモデルで精密に並べ替える」二段構え。大規模検索の定番。粗いふるい（ANN / 量子化）と相性が良い。

ゼロショット分類（再掲）

候補ラベルを文として埋め込み、対象ベクトルと最も近いものを選ぶ（第4章）。「検索」と同じ仕組みを「分類/タグ付け」に使う応用。

評価の目安

  • Recall@K — 上位 K に正解が含まれる割合。
  • Precision@K — 上位 K のうち正解の割合。
  • 検索 UX では「上位数件の質」が体感を左右する。
章群のまとめ 埋め込み（2章）→ Transformer（3章）→ CLIP（4章）で「意味のベクトル化」を、ベクトル検索（5章）・量子化（6章）で「速く軽く探す」を、LLM（7章）・融合（8章）で「前処理と手がかりの統合」を見た。これらの組合せが、語彙に縛られないオンデバイス写真検索を成立させている。

本アプリでの応用: オンデバイス AI（意味 + 構造化 + 字句を RRF 融合）
