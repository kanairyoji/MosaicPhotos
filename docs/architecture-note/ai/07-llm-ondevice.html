7. LLM とオンデバイス推論

LLM は「次のトークンを予測する」訓練を大規模に行った Transformer。端末で動かすには量子化・小型化と、推論の制約への配慮が要る。

LLM の基本動作

入力（プロンプト）をトークン列にし、モデルが次のトークンの確率分布を出す。1 つ選び、それを末尾に足してまた予測——を繰り返して文を生成する（自己回帰）。

graph LR P[プロンプト] --> TK[トークン列] TK --> M[Transformer] M --> NX[次トークンの確率] NX --> PICK[1つ選ぶ] PICK --> TK

用途は生成だけではない

  • 翻訳・要約・整形 — 入出力変換として使える。
  • 分類・抽出 — 「この文の意図は？」「日付を取り出して」。
  • 埋め込み生成 — 中間表現を文ベクトルとして使うモデルもある。

オンデバイスで動かす意味

観点クラウドオンデバイス
プライバシーデータが外部へ端末内で完結
通信必須・遅延あり不要・オフライン可
コストAPI 課金無料（端末資源のみ）
モデル規模巨大小型に制限

端末で動かす技術

  • 量子化 — 重みを int8/int4 などへ（→ 第6章）。サイズと速度を確保。
  • 専用ハード — モバイルの NPU/ANE で行列演算を高速化。
  • OS 提供モデル — OS 内蔵の基盤モデルを API 経由で使う（アプリが巨大モデルを同梱しなくてよい）。

制約と付き合う

  • メモリと発熱の上限がある。長い生成・大きな文脈は重い。
  • モデルが小さいぶん、難しい推論は不得手。小さく明確なタスクに絞ると安定する（例: 「クエリを英語に直す」）。
  • 結果は確率的。決定的な処理（検索条件の抽出など）はルールベースと併用すると堅牢。
実務の指針 端末 LLM は「巨大な万能 AI」ではなく「通信不要で軽い変換器」として使うのが現実的。検索の前処理（言語正規化・意図抽出）のような小さく定義された仕事に向く。
